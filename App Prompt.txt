You are a Senior Mobile Architect, Flutter Engineer, and Backend Integration Specialist.

Your task is to build a **production-ready Flutter mobile application (Android + iOS)** for this existing e-commerce platform.

The company already has:

• A fully functional **backend API**
• An **Admin dashboard**
• A **User web frontend** (production website)
• A **production database**

The mobile app must **reuse the same backend APIs and database**, so that any change made in the admin panel is automatically reflected in the mobile app.

The goal is to build a **high-performance, scalable, secure Flutter mobile app** that communicates with the existing backend and **visually matches the existing website UI/UX**.

---

PROJECT REQUIREMENTS

Build a **Flutter-based mobile application** that supports:

Android (APK + Play Store)
iOS (App Store)

Use **a single Flutter codebase** for both platforms.

The app must consume the existing backend APIs.

Do NOT modify backend logic unless required for mobile optimization.

---

SYSTEM ARCHITECTURE

Follow this architecture:

Admin Panel → Backend API → Database
Web Frontend → Backend API → Database
Flutter Mobile App → Backend API → Database

The Flutter mobile app must behave as another frontend client consuming the same API.

All data displayed in the app must come directly from the backend API.

---

IMPORTANT UI/UX REQUIREMENT (WEBSITE DESIGN SYNC)

The **mobile application UI/UX must match the existing user website**.

The mobile app must:

• Use the **same color palette as the website**
• Use the **same typography or closest mobile equivalent**
• Use the **same branding elements**
• Use the **same logos and icons**
• Follow the **same design language**

The UI should look like a **mobile-optimized version of the existing website**, ensuring brand consistency.

The agent should extract or replicate:

• Primary color
• Secondary color
• Background color
• Button styles
• Card styles
• Font family
• Brand assets

These should be implemented as **centralized theme configuration**.

Example structure:

lib/core/themes/app_theme.dart

The theme configuration must allow future updates to color schemes from a single location.

---

CONTENT SYNC WITH WEBSITE

The mobile app must display the **same content structure as the user website**, including:

Home banners
Featured products
Categories
Offers
Product descriptions
Pricing
Discounts

All content must be fetched from backend APIs to maintain real-time synchronization with the admin dashboard.
